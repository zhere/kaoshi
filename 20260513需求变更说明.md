# 国网后勤考试管理系统 - 2026-05-13 需求变更说明

## 变更背景

根据业务部门最新反馈，对考试管理系统的功能进行以下三项调整，以提升系统的易用性和管理效率。

---

## 变更一：用户管理功能增强

### 现状

当前用户管理仅支持手动单个添加用户，不支持批量导入，筛选条件较为基础（仅角色和部门）。

### 变更内容

#### 1.1 新增批量导入用户名单

| 项目 | 说明 |
|------|------|
| 功能位置 | 用户管理页面 → 用户管理 Tab |
| 入口 | 新增"导入用户"按钮，位于"添加用户"按钮右侧 |
| 导入格式 | 支持 Excel（.xlsx/.xls）和 CSV 文件 |
| 模板下载 | 提供标准导入模板（含用户名、姓名、角色、部门、电话、邮箱、工号字段） |
| 导入逻辑 | 解析文件后弹出预览确认框，展示待导入用户列表，确认后批量写入 |
| 重复校验 | 根据用户名判断是否已存在，若已存在可"跳过"或"覆盖" |
| 导入结果 | 弹窗展示成功/失败数量及失败原因 |

**导入模板字段设计：**

| 字段 | 必填 | 说明 |
|------|------|------|
| 用户名 | 是 | 登录账号，唯一标识 |
| 姓名 | 是 | 用户真实姓名 |
| 密码 | 否 | 默认为 123456 |
| 角色 | 是 | super_admin / admin / employee |
| 部门名称 | 是 | 需与系统中已存在的部门名称一致 |
| 联系电话 | 否 | - |
| 邮箱 | 否 | - |
| 员工工号 | 否 | 仅员工角色时有效 |

#### 1.2 新增条件筛选

在现有筛选（角色、部门、关键词搜索）基础上，新增以下筛选条件：

| 筛选条件 | 类型 | 说明 |
|----------|------|------|
| 用户状态 | 下拉选择 | 启用 / 禁用 |
| 创建时间范围 | 日期范围选择器 | 按用户创建时间筛选 |
| 高级搜索 | 折叠面板 | 支持多条件组合查询（用户名+姓名+角色+部门+状态+时间） |

#### 1.3 界面布局调整

```
┌─────────────────────────────────────────────────────────┐
│  用户管理                                                   │
├─────────────────────────────────────────────────────────┤
│  [+ 添加用户]  [+ 导入用户]  [下载模板]                     │
│  [角色 v] [部门 v] [状态 v] [创建时间  ──  ] [搜索...]   │
│  [展开高级搜索 ▸]                                         │
├─────────────────────────────────────────────────────────┤
│  ...表格数据...                                           │
└─────────────────────────────────────────────────────────┘
```

## 变更二：题目类目管理

### 现状

当前题目管理页面（[QuestionManagement.vue](file:///Users/xiaozhou/Desktop/诺心团队_华农/答题系统/src/views/admin/QuestionManagement.vue)）中已有一个"题目分类管理"弹窗（categoryDialogVisible），通过页面顶部的"分类管理"入口打开。当前实现为弹窗内嵌入表格，支持添加/编辑/删除分类。

### 分析结论：直接在题目管理模块内增强

**不建议新增独立模块**，原因如下：

1. 分类与题目是强关联关系，分离到独立模块会增加用户操作成本
2. 当前已有分类管理的基础实现，增强比重构更合理
3. 路由结构无需调整，减少变更范围

### 变更内容

#### 2.1 分类管理从弹窗改为内嵌 Tab 页

将现有的弹窗式分类管理升级为页面内的 Tab 页，与"题目列表"平级展示：

```
┌─────────────────────────────────────────────────────────┐
│  题目管理                                                   │
├─────────────────────────────────────────────────────────┤
│  [题目列表]  [类目管理]                                      │
└─────────────────────────────────────────────────────────┘
```

#### 2.2 类目管理 Tab 页功能

| 功能 | 说明 |
|------|------|
| 类目列表 | 表格展示：序号、类目名称、类目描述、题目数量、创建时间、状态（启用/禁用） |
| 添加类目 | 填写类目名称、描述、上级类目（支持多级分类树） |
| 编辑类目 | 修改类目名称、描述、上级类目 |
| 删除类目 | 删除前校验是否有题目关联，有关联则提示禁止删除或确认解除关联 |
| 启用/禁用 | 控制类目是否在选择题型时可见 |
| 类目排序 | 支持拖拽排序，影响前端展示顺序 |
| 批量操作 | 批量启用/禁用/删除 |

#### 2.3 类目数据结构

```javascript
{
  id: Number,
  name: String,              // 类目名称
  description: String,       // 类目描述
  parentId: Number | null,   // 上级类目ID（支持多级）
  sort: Number,              // 排序序号
  status: 'active' | 'disabled',
  createTime: String,
  updateTime: String
}
```

#### 2.4 对题目管理的影响

1. 题目编辑弹窗中的"分类"选择器改为树形选择（el-tree-select），支持多级类目
2. 题目列表的筛选条件中"分类"也升级为树形选择
3. 分类筛选支持"包含子类目"选项（选中父类目时自动包含子类目下的题目）

## 变更三：合并考试模板管理，增强考试管理

### 现状

当前管理端侧边栏包含两个独立菜单：
- **考试管理**（[ExamManagement.vue](file:///Users/xiaozhou/Desktop/诺心团队_华农/答题系统/src/views/admin/ExamManagement.vue)）— 考试的创建、发布、管理等
- **考试模板管理**（[ExamTemplateManagement.vue](file:///Users/xiaozhou/Desktop/诺心团队_华农/答题系统/src/views/admin/ExamTemplateManagement.vue)）— 模板的 CRUD 和关联设置

### 变更内容

#### 3.1 删除考试模板管理模块

- 从侧边栏菜单中移除"考试模板管理"入口
- 考试模板管理页面（ExamTemplateManagement.vue）及相关路由移除
- 侧边栏菜单顺序调整后为：用户管理 → 学习资料管理 → 考试管理 → 题目管理 → 积分管理 → 统计分析

#### 3.2 考试管理新增功能

##### 3.2.1 自定义题目组成配比

在创建/编辑考试的对话框中，新增"题目配比"配置区域，替代原有的"从题库选择"和"上传模板"方式。

**配置界面设计：**

```
┌────────────────────────────────────────────────────────┐
│  题目配比配置                                              │
├────────────────────────────────────────────────────────┤
│                                                         │
│  [类目一 v]  单选题 [─] 5 [+ ] 题  每题 [─] 5 [+ ] 分  │
│  [类目一 v]  多选题 [─] 3 [+ ] 题  每题 [─] 5 [+ ] 分  │
│  [类目一 v]  判断题 [─] 2 [+ ] 题  每题 [─] 5 [+ ] 分  │
│                                                         │
│  [+ 添加一行]                                            │
│                                                         │
│  ───────────────────────────────                         │
│  汇总：单选题 5 题 / 多选题 3 题 / 判断题 2 题 = 共 10 题 │
│  总分：50 分    及格分：[60] 分                          │
│                                                         │
│  ⚠ 提示：实际考试时从指定类目中随机抽取对应数量的题目     │
│                                                         │
└────────────────────────────────────────────────────────┘
```

**核心逻辑：**

| 配置项 | 说明 |
|--------|------|
| 类目 | 从题目类目树中选择，支持多级类目 |
| 题型 | 单选题 / 多选题 / 判断题 |
| 题目数量 | 从该类目中抽取的该题型数量 |
| 每题分值 | 该题型的单题分值 |
| 及格分数 | 自动根据总分计算，可手动调整 |

**数据模型变化：**

```javascript
// 考试模型的 questionConfig 字段替代原有的 questions 字段
{
  // ... 原有考试字段
  questionConfig: [
    {
      categoryId: Number,     // 类目ID
      type: 'single' | 'multiple' | 'judge',
      count: Number,          // 抽取数量
      scorePerQuestion: Number // 每题分值
    }
  ],
  // 移除原有 questions: Number[] 字段
}
```

**考试时的题目生成逻辑：**

```
创建考试时记录配比规则
        ↓
发布考试时根据配比规则从题库随机抽题
  - 按类目+题型筛选符合条件的题目池
  - 从池中随机抽取指定数量的题目
  - 若题目池不足，提示管理员补充题目
        ↓
抽题结果固定，所有考生考试题目一致（统一试卷）
```

**新增配置项：是否开启随机抽题**

- **统一试卷**（默认）：发布时一次性抽题，所有考生题目相同
- **随机抽题**：每位考生进入考试时独立抽题，题目不同但配比相同

##### 3.2.2 支持编辑考试对象（指定部门）

在创建/编辑考试的对话框中，将现有的"考试权限"区域增强为"考试对象"配置。

**配置界面设计：**

```
┌────────────────────────────────────────────────────────┐
│  考试对象配置                                              │
├────────────────────────────────────────────────────────┤
│                                                         │
│  ○ 全员可参加                                              │
│  ○ 指定部门参加          [技术部 ✕] [运维部 ✕] [+ 添加]  │
│  ○ 指定人员参加          [王五 ✕] [赵六 ✕] [+ 添加]      │
│  ○ 按岗位/角色参加       [管理员 ✕] [员工 ✕] [+ 添加]     │
│                                                         │
│  ● 可见范围                                              │
│  ☑ 未到开始时间时可见（若关闭，未开始时考生看不到考试）   │
│                                                         │
│  ● 参加限制                                              │
│  允许参加次数：[1 次]  (1-99)                             │
│  ☑ 必须先完成关联学习资料                                │
│                                                         │
└────────────────────────────────────────────────────────┘
```

**新增字段：**

```javascript
{
  // ... 原有考试字段
  targetType: 'all' | 'department' | 'user' | 'role',  // 考试对象类型（扩展原有 permissionType）
  targetDepartments: Number[],    // 指定部门ID列表（扩展原有 allowedDepartments）
  targetUsers: Number[],          // 指定人员ID列表（扩展原有 allowedUsers）
  targetRoles: String[],          // 指定角色列表（新增）
  visibleBeforeStart: Boolean,    // 开始前是否可见（新增）
  maxAttempts: Number,            // 最大参加次数（新增），默认1次
  requireMaterialCompletion: Boolean // 是否必须先完成关联资料（新增）
}
```

#### 3.3 考试状态流转更新

```
draft（草稿）──── 编辑配置 ────→ ready（待发布）
  ↑                                   │
  │           ┌── 发布（生成海报）──┐  │
  │           ▼                    ▼  │
  │      ongoing（进行中）   推广海报下载 │
  │           │                       │
  │        暂停/恢复                   │
  │           │                       │
  │        ended（已结束）             │

```

新增"草稿"状态：创建考试后默认为草稿，配置完善后才可发布。

#### 3.4 考试列表新增列

| 列名 | 说明 |
|------|------|
| 考试对象 | 显示"全员"/"指定部门（技术部等）"/"指定人员"/"按角色" |

## 开发建议

### 优先级排序

| 优先级 | 变更项 | 理由 |
|--------|--------|------|
| P0 | 3.2 题目配比 | 核心业务变更，影响考试创建流程 |
| P0 | 3.1 删除模板管理 | 需要先合并能力再删除旧模块 |
| P1 | 2. 类目管理 | 题目配比依赖类目体系，需前置或并行 |
| P1 | 3.3 考试对象 | 与考试编辑表单耦合，建议同步开发 |
| P2 | 1.1 导入用户 | 独立功能，可并行开发 |
| P2 | 1.2 条件筛选 | 简单增强，可快速完成 |

### 建议开发顺序

```
第一轮（P0）：类目管理 + 题目配比 + 删除模板管理
  → 建立类目体系 → 考试支持配比配置 → 下线模板模块

第二轮（P1）：考试对象增强
  → 扩展考试权限配置能力

第三轮（P2）：用户导入 + 筛选增强
  → 可独立并行开发
```

## 附录：数据迁移说明

删除考试模板管理后，现有的考试模板数据建议做如下处理：

| 处理方式 | 说明 |
|----------|------|
| 已有模板保留 | 已创建的考试不受影响 |
| 模板数据归档 | 在 mockData.js 中保留模板数据用于历史查询 |
| 新建考试不再使用模板入口 | 考试创建界面移除"选择模板"步骤，改为直接配置配比 |
| 兼容处理 | 若旧考试使用模板创建，在详情中显示"由模板XXX创建"的记录信息 |
