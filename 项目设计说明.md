# 国网后勤考试管理系统 - 项目设计说明

## 一、项目概述

国网后勤考试管理系统是一个面向国家电网后勤部门的在线考试与学习管理平台，提供从考试创建、题目管理、在线答题到成绩统计的完整闭环功能，同时整合学习资料管理与积分激励体系，支持管理端和员工端双门户访问。

### 1.1 项目定位

- **管理端**：为管理员提供考试全生命周期管理、题库管理、用户管理、数据统计等后台操作能力
- **员工端**：为一线员工提供在线考试、学习资料查阅、积分查询、历史记录等参与功能

### 1.2 核心业务流程

```
管理员创建考试 → 配置题目/时间/权限 → 发布考试（自动生成推广海报） → 
员工扫码/登录 → 在线答题 → 自动判分 → 查看成绩/错题 → 学习资料 → 获取积分
```

### 4.2 导航守卫逻辑

在路由的 `beforeEach` 守卫中实现三层验证：

1. **登录验证**：需认证的路由检查 `userStore.isLoggedIn`
2. **角色验证**：管理端路由检查 `loginType === 'admin'`
3. **自动跳转**：已登录用户访问 `/login` 时自动跳转到对应首页

## 六、管理端业务逻辑

### 6.1 登录模块 (`Login.vue`)

- 支持管理端和员工端两种登录入口选择
- 基于 mock 数据验证用户名和密码
- 角色校验：员工账号禁止登录管理端，管理员账号禁止登录员工端
- 登录成功后跳转至对应首页

### 6.2 用户管理 (`UserManagement.vue`)

**三个Tab页：**

| Tab | 功能 |
|-----|------|
| 用户管理 | 用户 CRUD、角色筛选、部门筛选、密码重置、权限设置 |
| 部门管理 | 部门 CRUD，含上级部门、部门负责人 |
| 角色管理 | 角色 CRUD，含权限树分配 |

**数据结构：**
- 用户字段：用户名、姓名、角色（super_admin/admin/employee）、部门、电话、邮箱、状态
- 部门字段：名称、上级部门、负责人、成员数量
- 角色字段：名称、代码、描述、用户数

**权限树**：通过 `el-tree` 组件展示权限层级，权限按模块划分为用户管理、考试管理、题目管理、学习资料管理、统计分析五个模块，每个模块含查看/添加/编辑/删除等细粒度权限。

### 6.3 题目管理 (`QuestionManagement.vue`)

**功能点：**
- 题目 CRUD（单选题/多选题/判断题）
- 按分类、类型、难度筛选，关键词搜索
- 选项动态增删（支持2-6个选项）
- Excel/CSV 批量导入题目
- 下载导入模板
- AI 自动生成考题（模拟实现）
- 题目分类管理

**题目模型：**
```javascript
{
  id: Number,
  type: 'single' | 'multiple' | 'judge',
  category: Number,      // 分类ID
  difficulty: 'easy' | 'medium' | 'hard',
  content: String,       // 题目内容
  options: String[],     // 选项列表
  answer: String,        // 正确答案
  analysis: String,      // 答案解析
  createTime: String
}
```

### 6.4 考试管理 (`ExamManagement.vue`)

**核心功能：**

| 功能 | 说明 |
|------|------|
| 创建考试 | 从模板创建或自定义，配置名称、时间、时长、题目、分数、权限 |
| 编辑考试 | 修改考试配置 |
| 发布考试 | 生成推广海报（含二维码），供管理员下载分享 |
| 暂停/重启 | 控制考试状态流转 |
| 复制考试 | 基于已有考试快速创建副本 |
| 删除考试 | 删除考试记录 |
| 考试详情 | 查看考试配置和统计数据 |
| 上传模板 | Excel/CSV 导入题目 |

**状态流转：**
```
upcoming（未开始）───发布──→ ongoing（进行中）───暂停──→ paused（暂停）
    ↑                        ↑                          │
    └──────────── 重启 ──────┴────────── 重启 ──────────┘
```

**发布海报功能：**
- 使用 Canvas 绘制 750×1200px 的推广图片
- 图片包含：国网绿色主题、考试名称、考试时间、时长、题量、及格分等信息
- 使用 QRCode 库生成考试二维码嵌入图片
- 底部品牌标识"国网后勤考试系统"
- 管理员可下载 PNG 图片分享给员工

### 6.5 考试模板管理 (`ExamTemplateManagement.vue`)

**功能点：**
- 模板 CRUD（含题目分类、难度、题量、时长、及格分）

- 查看模板下的考试列表

- 上传 Excel/CSV 模板文件

**模板与考试的关系：** 一对多关系，一个模板可创建多个考试。

### 6.6 学习资料管理 (`MaterialManagement.vue`)

**两个Tab页：**

| Tab | 功能 |

**资料类型：** PDF、视频、图片、文档

**文件上传：** 通过 `el-upload` 拖拽上传，支持 PDF/MP4/JPG/DOC 等格式。

## 七、员工端业务逻辑

### 7.1 考试列表 (`ExamList.vue`)

- 展示可参与的考试（过滤已结束的考试）
- 卡片式布局，显示考试名称、时长、题量、总分、及格分、时间范围
- 支持按状态筛选和关键词搜索
- 点击考试查看详情，可跳转开始答题

### 7.2 在线答题 (`OnlineExam.vue`)

**核心功能：**
- 逐题作答，支持单选题/多选题/判断题
- 题目导航按钮（已答绿色高亮、标记的黄色高亮）
- 标记题目功能
- 倒计时（基于考试时长自动计时）
- 提交确认对话框（显示已答/未答/标记数）

**考试流程：**
```
进入考试 → 开始计时 → 逐题作答 → 提交确认 → 自动判分 → 跳转结果页
```

**判分逻辑：**
```javascript
// 逐题对比用户答案与正确答案
const userAnswerStr = Array.isArray(userAnswer) ? userAnswer.sort().join('') : userAnswer
if (userAnswerStr === correctAnswer) {
  correctCount++
}
const score = Math.round((correctCount / totalQuestions) * 100)
```

### 7.3 考试结果 (`ExamResult.vue`)

- 成绩展示（圆形得分组件，及格绿色/不及格红色）
- 正确/错误/总题数统计
- 重新答题功能
- 答题详情：按全部/正确/错误筛选
- 逐题展示：用户答案、正确答案、答案解析
- 选项颜色标记（绿色正确、红色错误）

### 7.4 历史记录 (`HistoryRecord.vue`)

**两个板块：**

| 板块 | 功能 |

错题本功能：自动收集所有考试中的错题，支持按考试分组查看正确答案和解析。

### 7.5 学习资料 (`MaterialList.vue`)

- 每个考试组显示学习进度条

- 资料预览弹窗（PDF/视频/图片占位页面）

### 7.6 我的积分 (`MyPoints.vue`)

**三个板块：**

| 板块 | 内容 |

### 13.4 异常与边界场景

#### 场景十八：账号与登录异常处理

| 异常场景 | 处理方式 |
|----------|---------|
| 忘记密码 | 员工端点击"忘记密码" → 提示联系管理员重置 |
| 账号被禁用 | 登录提示"账号已被禁用，请联系管理员" |
| 登录设备更换 | 支持多设备登录，但同一账号不能同时在多端答题 |
| 账号过期 | 管理员可在后台设置账号有效期，过期后禁止登录 |

#### 场景十九：考试过程异常处理

| 异常场景 | 处理方式 |
|----------|---------|
| 答题中断网 | 本地缓存答题记录 → 恢复网络后自动提交（建议5分钟内恢复） |
| 答题过程中退出 | 再次进入考试 → 自动恢复到上次答题进度（未提交情况下） |
| 超时未交卷 | 系统自动强制提交当前已答内容，已答部分正常判分 |
| 设备故障/白屏 | 刷新页面 → 恢复答题进度（基于本地缓存） |
| 切屏检测 | 检测到切出考试页面（超过3次/超过N秒）→ 记录并警告 → 强制交卷 |
| 多设备同时登录 | 同一账号在另一设备登录 → 当前设备提示"账号在其他设备登录" |

#### 场景二十：考试配置异常处理

| 异常场景 | 处理方式 |
|----------|---------|
| 某类目下题目不足 | 发布考试时校验：若某类目+题型的题目池数量不满足配比要求，提示管理员补充 |
| 考试时间冲突 | 创建考试时检测：同一批考试对象在同一时间段是否有其他考试（仅预警，不强制） |
| 无权限员工扫码 | 提示"您无权参加该考试，请联系管理员" |
| 超过参加次数 | 提示"已达到最大参加次数" |
| 考试结束后扫码 | 提示"考试已结束" |

#### 场景二十一：数据安全与合规场景

| 场景 | 处理方式 |
|------|---------|
| 敏感数据脱敏 | 导出员工信息时，手机号中间4位脱敏显示（138****0000） |
| 操作日志记录 | 管理员所有关键操作（创建/修改/删除）记录操作日志，可追溯 |
| 数据备份 | 每日自动备份考试记录和用户数据（生产环境） |
| 隐私合规 | 员工个人信息仅管理员可见，员工之间不可见他人信息 |
| 防作弊机制 | 随机选项顺序（同一考试不同考生的选项顺序不同） |

### 13.5 多端协同场景

#### 场景二十二：管理端PC + 员工小程序/H5 协同

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│        管理端（PC Web）       │     │    员工端（小程序/H5）      │
├─────────────────────────────┤     ├─────────────────────────────┤
│  用户管理（增删改查/导入）   │────→│  登录/注册                  │
│  题目管理（上传/编辑/导入）  │     │  参加考试（答题/提交）       │
│  考试管理（创建/发布/监控）  │     │  查看结果（成绩/解析）       │
│  学习资料管理（上传/关联）   │────→│  学习资料（在线阅读/视频）   │
│  积分管理（规则/调整）       │←────│  积分查询（明细/排行）       │
│  统计分析（图表/报表）       │←────│  历史记录（错题本）          │
│  数据导出（Excel/CSV）       │     │  消息通知                   │
│  系统配置（部门/角色/权限）  │     │  个人中心（账号/密码）       │
└─────────────────────────────┘     └─────────────────────────────┘
```

**数据流向：**
- 管理员在管理端创建配置 → 数据写入服务端 → 员工端读取展示
- 员工在员工端操作（答题/学习） → 数据写入服务端 → 管理端统计展示
- 二维码作为连接纽带：管理端生成 → 员工扫码 → 跳转员工端入口
